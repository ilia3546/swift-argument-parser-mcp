# Code of Conduct

This project follows a minimal code of conduct, in the same spirit as the rest
of the repository: short, practical, and applied in good faith.

## Our standards

We want this project to be a place where people can collaborate on code without
friction. To keep it that way, contributors and maintainers are expected to:

- Be civil. Assume good faith.
- Critique code, not people. Keep feedback specific, technical, and actionable.
- Stay on topic. Issues, pull requests, and discussions are about the library
  and how to improve it.
- Respect maintainers' time. Search before filing, follow the templates, and
  give reviewers room to respond.
- Welcome newcomers. Everyone was new once.

## Unacceptable behaviour

The following are not welcome in any project space (issues, pull requests,
discussions, review comments, commit messages):

- Personal attacks, insults, or demeaning remarks.
- Harassment, intimidation, or sustained disruption of conversation.
- Discrimination on the basis of any personal characteristic.
- Publishing someone else's private information without their consent.

## Scope

This code of conduct applies to all project spaces — the GitHub repository,
its issues and pull requests, GitHub Discussions, and any communication that
takes place under the project's name.

## Reporting

If you experience or witness behaviour that violates this code of conduct,
contact the maintainers directly via the contact details on their GitHub
profile. Reports will be handled confidentially. Maintainers reserve the right
to remove comments, close issues or pull requests, and block accounts that
repeatedly violate this code.
